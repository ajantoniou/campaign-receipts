# Higgsfield — video & image generation (portfolio)

**Higgsfield** ([higgsfield.ai](https://www.higgsfield.ai)) is an **AI video and image** platform (consumer through **enterprise**): long-form and short-form video, motion control, marketing-focused tooling (e.g. ads, UGC-style workflows), and collaboration features for teams.

**About / positioning:** [About Higgsfield](https://higgsfield.ai/about) · **Enterprise:** [Enterprise hub](https://higgsfield.ai/enterprise)

Capabilities evolve quickly (multiple models and “studios”); treat this skill as **workflow + governance**, not a feature checklist.

## When to use it here

- **NT Ministry / Video Producer** personas: sermon shorts, trailers, faceless-safe workflows — align with **theology + compliance** gates in company `AGENTS.md` before publishing.
- **Marketing / growth**: paid social, landing-page loops — pair exports with **brand** approvals from **design** (Claude Design / Open Design) so visuals stay on-brand.
- **Evaluation**: portfolio backlog may compare **Higgsfield vs Runway / Sora / Kling** — record **cost, TOS, API fit, and compliance** in issues; no paid signup or keys without founder approval.

## Workflow

1. **Brief** in Paperclip / issue: audience, aspect ratio, duration cap, **do-not-show** list (faces, minors, disputed claims).
2. **Generate** in Higgsfield; keep **project links** and **asset IDs** in the issue (not secrets).
3. **Post-production**: VO (e.g. ElevenLabs), edit (e.g. DaVinci) per existing video pipeline docs where applicable.
4. **Ship**: upload to approved channel; store **final** files in company-approved storage — not only inside Higgsfield.

## Credentials & spend

- **API keys, team seats, and billing** live in **founder-approved** vaults or Higgsfield org settings — **never** commit keys or paste into repo issues verbatim.
- Follow `.cursor/rules/credentials-and-push-safety.mdc`; treat platform logins like any other production credential.
